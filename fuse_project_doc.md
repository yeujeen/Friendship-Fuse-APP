# Friendship FUSE 프로젝트 기획서

---

## ❗ PRD (Product Requirements Document)

### **프로젝트 개요**

- **프로젝트 명:** Friendship FUSE (프렌드십 퓨즈)

- **핵심 컨셉:** AI가 두 사람의 음악 취향을 분석해서 "우리가 같이 들을 노래"를 찾아주는 '연결', '발견', '공유'의 소셜 음악 앱

- **핵심 목표:** 
  - 친구들끼리 음악 취향 공유할 때 느끼는 부담감 해소
  - 두 사람이 함께 들을 수 있는 플레이리스트를 AI가 자동으로 만들어줌
  - 음악을 통해 친구관계가 더 깊어지는 경험 제공

- **타겟 사용자 (누가 쓸까?):**
  - 20~30대 대학생 및 직장인
  - 친구들과 음악 공유를 즐기지만 "내 취향이 이상하면 어쩌지?" 걱정하는 사람
  - 팀플, 스터디, 드라이브 등 함께 음악 들을 일이 많은 사람
  - 새로운 음악 발견하는 걸 좋아하는 사람

- **왜 이 앱이 필요해?:**
  - **문제 1**: 친구한테 내 플레이리스트 공유하기 부담스러워요 (취향이 너무 달라서)
  - **문제 2**: 같이 들을 노래 고르는데 시간이 너무 오래 걸려요
  - **문제 3**: 서로 모르는 노래만 나와서 재미없어요
  - **해결책**: AI가 "둘 다 좋아할 만한 노래"를 자동으로 찾아줍니다!

## ❗ 상세 기능 명세

### **P0 (Core Features - 꼭 만들어야 하는 핵심 기능)**

#### **1. 음악 취향 입력 기능**
- **[사용자가 뭘 해?]** 내가 좋아하는 가수/아티스트 이름을 5개 입력합니다.
- **[어떻게 동작해?]**
  - 메인 화면에서 "좋아하는 아티스트 입력" 칸이 보임
  - 한 명씩 차례로 입력 (예: "아이유", "뉴진스", "테일러 스위프트", "데이식스", "10cm")
  - 입력한 데이터는 서버로 전송됨
  - API: `POST /api/user-input`

#### **2. 친구 초대 및 데이터 연결**
- **[사용자가 뭘 해?]** 친구에게 링크를 보내서 친구도 자기 취향을 입력하게 합니다.
- **[어떻게 동작해?]**
  - "친구 초대하기" 버튼 클릭 → 공유 링크 생성
  - 친구가 링크 들어와서 자기 아티스트 5개 입력
  - 두 사람 데이터가 준비되면 자동으로 분석 시작
  - API: `POST /api/invite`, `GET /api/session/:id`

#### **3. 취향 유사도 분석 (AI 기능)**
- **[사용자가 뭘 봐?]** "우리 둘의 음악 취향은 45% 비슷해요!" 같은 결과를 봅니다.
- **[어떻게 동작해?]**
  - Google Gemini API에 두 사람의 데이터를 전송
  - AI가 장르, 분위기, 아티스트 스타일 등을 비교 분석
  - 유사도 % 계산 (자카드 유사도 방식)
  - **벤 다이어그램**으로 시각화: 
    - 왼쪽 원 = 나만 듣는 음악
    - 오른쪽 원 = 친구만 듣는 음악
    - 겹치는 부분 = 우리 둘 다 좋아할 음악
  - API: `POST /api/analyze` (Gemini API 호출)

#### **4. FUSE Mix 자동 생성**
- **[사용자가 뭘 봐?]** "우리 둘을 위한 플레이리스트" 10~15곡이 자동으로 만들어져 있습니다.
- **[어떻게 동작해?]**
  - AI가 두 사람의 공통 취향 영역에서 노래를 선택
  - 플레이리스트 제목도 AI가 자동 생성 (예: "팀플 집중 모드 믹스")
  - 각 곡마다 "왜 이 노래를 골랐는지" 설명 문구 제공
  - API: `GET /api/fuse-mix/:sessionId`

#### **5. 결과 공유 기능**
- **[사용자가 뭘 해?]** 결과 화면을 SNS에 공유할 수 있습니다.
- **[어떻게 동작해?]**
  - "공유하기" 버튼 → 고유 URL 생성
  - 카카오톡, 인스타그램 등으로 공유 가능
  - 공유받은 사람도 결과 확인 가능 (단, 수정은 불가)
  - API: `GET /api/share/:mixId`

## ❗ UIUX 요구사항

### **디자인 철학: "친근하고 재미있게 (Friendly & Fun)"**

음악은 즐거운 거니까, 앱도 보는 것만으로 기분 좋아야 해요!

#### **1. 색상 (Color Palette)**
- **메인 컬러:** 밝고 에너제틱한 그라데이션
  - 보라색 → 분홍색 → 주황색 그라데이션
  - 음악의 다채로움을 표현
- **배경:** 깨끗한 흰색 또는 연한 회색
- **강조 색:** 포인트 버튼에 밝은 노란색 또는 민트색

#### **2. 폰트 (Typography)**
- **제목:** 굵고 임팩트 있는 산세리프체 (예: Pretendard Bold)
- **본문:** 읽기 편한 고딕체
- **플레이리스트 제목:** 약간 손글씨 느낌의 친근한 폰트

#### **3. 레이아웃 및 인터랙션**
- **심플한 3단계 플로우:**
  1. 입력 화면 (아티스트 입력)
  2. 분석 중 화면 (로딩 애니메이션)
  3. 결과 화면 (유사도 + 플레이리스트)
- **벤 다이어그램:** 두 원이 겹치는 모습을 크게, 직관적으로
- **버튼:** 둥근 모서리, 눌렀을 때 통통 튀는 애니메이션
- **공유 버튼:** 눈에 확 띄게 하단 고정

#### **4. 방해 요소 최소화**
- 복잡한 메뉴 없음 (3단계만 집중)
- 광고 없음 (초기 버전)
- 불필요한 효과음 없음 (결과 나올 때 짧은 효과음만)

#### **5. 모바일 우선 (Mobile-First)**
- 스마트폰에서 사용하기 편하게 설계
- 한 손으로도 쉽게 입력 가능
- 세로 모드 최적화

### **핵심 API 엔드포인트 (서버가 받는 요청들)**

| 요청 종류 | 주소 | 설명 |
|---------|------|------|
| POST | /api/user-input | 사용자가 아티스트 입력 |
| POST | /api/invite | 친구 초대 링크 생성 |
| GET | /api/session/:id | 세션 데이터 가져오기 |
| POST | /api/analyze | Gemini AI 분석 시작 |
| GET | /api/fuse-mix/:id | 플레이리스트 결과 보기 |
| GET | /api/share/:mixId | 공유용 링크 생성 |

---

### **데이터 구조 (어떤 정보를 다루나?)**

**입력 데이터:**
```
사용자 A: ["아이유", "뉴진스", "데이식스", "10cm", "볼빨간사춘기"]
사용자 B: ["아이유", "르세라핌", "데이식스", "IU", "방탄소년단"]
```

**분석 결과:**
```
유사도: 45%
공통 요소: ["K-pop", "발라드", "감성적", "아이유", "데이식스"]
```

**FUSE Mix:**
```
제목: "우리만의 감성 팝 타임"
곡 리스트: [곡1, 곡2, ... 곡10]
생성 이유: "두 분 모두 감성적이고 멜로디가 좋은 음악을 선호하시네요!"
```
